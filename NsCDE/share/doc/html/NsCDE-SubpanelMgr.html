4.11. Subpanel Manager

Only in NsCDE. Tool written for managing FvwmButtons transient Subpanels which are opened from the Front Panel. It is called from Install Icon action on every Subpanel. This tool exists because FvwmButtons doesn't implement drag and drop, and there is no application manager present, since this part of CDE functions cannot be easy implemented even if some file manager is taken to act as application manager.

Subpanel Manager has list of all applications which are providing system menu with it's presence (patched version of fvwm-menu-desktop is used in the background), and list of applications provided in the current Subpanel from which tool is called by "Install Icon". This lists are displayed on the top of the window side by side: system menu applications on the left, and current Subpanel items on the right list. There are 3 text fields: Name, Command and Icon File. This fields are automatically populated when some item from mentioned lists is clicked, but it can also be populated manually if user wishes to add a custom application, Fvwm Module, Fvwm Function or other entry manually on the Subpanel. Subpanel Manager can also remove existing entries from the Subpanel or modify them.

Subpanel Manager implements this helper functions for modifying and applying settings on new items:

  • Launcher Type

  • Execute in Terminal

  • Do not check for command existence

  • Icon indicator

This tool is not well designed and will be rewritten on the first occasion. Subpanel configuration files NsCDE-SubpanelX.conf, where X is 1-10 can be edited by hand in $FVWM_USERDIR if order of entries is important, or something needs to be changed on existing entries.
